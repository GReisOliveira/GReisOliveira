# 🌐 About RESTful APIs:

I use REST (Representational State Transfer) architecture to build efficient, standardized, and scalable APIs, applying fundamental concepts such as:

*   **Statelessness:** Each client request contains all the necessary information to be processed independently.
*   **Standardized HTTP Methods:** Proper use of verbs like `GET` (filtering and pagination), `POST` (creation and JWT authentication), `PUT`/`PATCH` (updates), and `DELETE` (resource removal).
*   **HTTP Status Codes:** Clear semantic returns for the client (e.g., `200`, `201`, `400`, `401`, `404`).
*   **Decoupling (Separation of Concerns):** Ensuring that the back-end (FastAPI) functions independently of the web client (React), communicating strictly through structured JSON.
