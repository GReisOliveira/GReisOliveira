### 🌐 Sobre APIs RESTful

Em meus projetos, utilizo a arquitetura **REST (Representational State Transfer)** para construir APIs eficientes, padronizadas e escaláveis. Aplico os principais conceitos do modelo, como:

* **Statelessness (Ausência de Estado):** Cada requisição do cliente contém todas as informações necessárias para ser processada, sem depender de sessões armazenadas no servidor.
* **Métodos HTTP Padronizados:** Utilização correta dos verbos HTTP para manipulação de recursos:
  * `GET` para buscar dados (ex: paginação dinâmica e filtros de busca).
  * `POST` para criação (incluindo operações de *Bulk Insert* e autenticação via Token JWT).
  * `PUT`/`PATCH` para atualizações.
  * `DELETE` para remoção.
* **Códigos de Status HTTP:** Retornos claros para o cliente, mapeando erros (ex: `400`, `401`, `404`) e sucessos (ex: `200`, `201`) de forma semântica.
* **Separação de Responsabilidades (Decoupling):** Garantia de que o back-end (FastAPI) funcione de forma totalmente independente do cliente web (React), comunicando-se estritamente através de JSON estruturado.
