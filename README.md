# 🌐 Sobre APIs RESTful:

Utilizo a arquitetura **REST (Representational State Transfer)** para construir APIs eficientes, padronizadas e escaláveis, aplicando conceitos fundamentais como:

* **Statelessness (Ausência de Estado):** Cada requisição do cliente contém todas as informações necessárias para ser processada de forma independente.
* **Métodos HTTP Padronizados:** Utilização correta dos verbos `GET` (filtros e paginações), `POST` (criações e autenticação JWT), `PUT`/`PATCH` (atualizações) e `DELETE` (remoção de recursos).
* **Códigos de Status HTTP:** Retornos semânticos claros para o cliente (ex: `200`, `201`, `400`, `401`, `404`).
* **Decoupling (Separação de Responsabilidades):** Garantia de que o back-end (FastAPI) funcione de forma independente do cliente web (React), comunicando-se estritamente através de JSON estruturado.
